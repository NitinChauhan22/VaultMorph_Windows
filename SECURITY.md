# Security Policy

VaultMorph takes security seriously. We appreciate responsible disclosure of vulnerabilities.

---

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 1.0.x   | Yes       |
| <1.0    | No        |

---

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

**Do NOT open a public GitHub issue.**

Instead, contact:

📧 Email: [resecureit@gmail.com](mailto:resecureit@gmail.com)

Please include:

* Description of the vulnerability
* Steps to reproduce
* Potential impact
* Proof of concept (if available)

---

## Response Timeline

We aim to respond promptly to all security reports:

* Initial acknowledgment: within **48 hours**
* Status update: within **5 days**
* Resolution timeline: depends on severity and complexity

---

## Responsible Disclosure

We request that you:

* Do not publicly disclose vulnerabilities before a fix is released
* Allow reasonable time for remediation
* Avoid exploiting the vulnerability beyond proof-of-concept

We will notify you once the issue has been resolved.

---

## Scope

This policy applies to:

* VaultMorph desktop application
* Encryption and key management mechanisms
* Local storage and vault handling

Out of scope:

* Issues caused by compromised user devices
* Misconfiguration or unsafe user practices
* Third-party libraries (report to respective maintainers)

---

## Security Commitment

VaultMorph is designed with a strong focus on:

* Local-first data security
* Industry-standard encryption practices
* Zero telemetry and data collection

We continuously work to improve the security of the application.
